--[[
Абстрактный рабочий пример для Arizona Mobile:

-- Получаем координаты игрока
local playerX, playerY, playerZ = getCharCoordinates(PLAYER_PED)

-- Координаты целевого объекта (пример: координаты дома)
local objX, objY, objZ = 123.4, 567.8, 910.1

-- ВАЖНО: convert3DCoordsToScreen работает ТОЛЬКО если эти функции определены в MoonLoader
local startX, startY = convert3DCoordsToScreen(playerX, playerY, playerZ)
local endX, endY = convert3DCoordsToScreen(objX, objY, objZ)

-- Проверяем, видны ли обе точки на экране
if startX and endX then
    -- Рисуем линию от игрока к объекту
    renderDrawLine(startX, startY, endX, endY, 0xFF2196F3, 2)  -- Синяя линия, толщина 2px
else
    -- Если точки вне экрана, ничего не рисуем
    print("[ESP] Объект вне зоны видимости")
end

РЕЗУЛЬТАТ:
✓ На экране появится синяя линия от игрока к объекту
✓ Если объект вне видимости - линия не рисуется (оптимизация)
✓ Система автоматически масштабируется под разрешение экрана
]]

═══════════════════════════════════════════════════════════════════════════

--[[
ПРОБЛЕМА: Скрипт не загружается
РЕШЕНИЕ:
  1. Проверьте, что файл main.lua находится в папке moonloader/
  2. Убедитесь, что папка lua/ находится в moonloader/
  3. Перезагрузите игру
  4. Проверьте консоль MoonLoader (F6) на ошибки

ПРОБЛЕМА: Меню не открывается (/mmenu не работает)
РЕШЕНИЕ:
  1. Проверьте, что UIFramework инициализирован: UIFramework:initialize()
  2. Убедитесь, что mobile_ui_framework.lua находится в lua/
  3. Введите /uiinfo для проверки состояния

ПРОБЛЕМА: ESP система не работает
РЕШЕНИЕ:
  1. Проверьте, что esp_system.lua находится в lua/
  2. Убедитесь, что ESPSystem инициализирован
  3. Введите /espstats для диагностики

ПРОБЛЕМА: Линии не рисуются
РЕШЕНИЕ:
  1. Проверьте, что convert3DCoordsToScreen доступна в вашей версии MoonLoader
  2. Убедитесь, что объекты в пределах maxDistance (обычно 500м)
  3. Введите /esplines для включения линий
  4. Проверьте, что renderDrawLine поддерживается
]]

--[[
РЕКОМЕНДАЦИИ ДЛЯ ARIZONA MOBILE:

1. МАСШТАБИРОВАНИЕ:
   • На смартфонах обычно разрешение 1080x1920
   • На планшетах может быть 1440x2560 или выше
   • Система автоматически масштабируется

2. ПРОИЗВОДИТЕЛЬНОСТЬ:
   • Не создавайте более 1000 объектов в ESP одновременно
   • Используйте интервалы обновления 100-200мс
   • Ограничивайте maxDistance до 500м

3. ОПТИМАЛЬНЫЕ НАСТРОЙКИ:
   UIFramework.config.menu_width = 280      -- для мобильных экранов
   ESPSystem.config.maxDistance = 300       -- оптимальное расстояние
   ESPSystem.config.updateInterval = 150    -- хороший баланс

4. ПОТРЕБЛЕНИЕ ПАМЯТИ:
   • UI Framework: ~2-3 MB
   • ESP System: ~5-10 MB (зависит от кол-ва объектов)
   • Итого: ~10-15 MB при нормальной работе

5. БАТАРЕЯ:
   • Уменьшите updateInterval для экономии батареи
   • Отключите линии, если не нужны
   • Используйте ESP выборочно
]]
